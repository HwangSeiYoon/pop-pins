## 📋 MVP 완성 및 발표 준비 To-Do List

### 1️⃣ MVP 완성 및 품질 검증

-  **핵심 기능 완성**
    
    - 최소 1~2개 핵심 기능(AC 수용 기준 충족)
        
    - 실제로 동작 가능한 상태로 구현
        
-  **품질 테스트**
    
    - 주요 시나리오별 정상/예외 플로우 검증
        
    - 로딩 속도, 오류율, 안정성 테스트
        
-  **운영 환경 점검**
    
    - 로그 수집, 모니터링, 이벤트 핸들링, SLA 최소 수치 확보
        

---

### 2️⃣ 데모 준비

-  **데모 시나리오 스크립트 작성**
    
    - “사용자 → 문제 → 해결 → 가치 전달” 흐름을 중심으로 구성
        
    - 단계별 클릭/입력/예상 반응/Fallback 포함
        
-  **라이브 데모 리허설**
    
    - 실제 사용자 시나리오 기준 90~120초 이내 데모 구성
        
    - 문제 해결 후 가치 전달이 명확히 보이도록 연출
        
-  **데모 영상 백업 준비**
    
    - 네트워크 불안 대비용 영상 녹화본 확보
        
    - “Fallback 플랜” 포함 (영상 시연 or 로컬 실행 대체안)
        

---

### 3️⃣ 발표자료(PPT) 제작

-  **발표 구조 구성**
    
    - 문제 → 해결(MVP) → 시연 → 성과/KPI → 가정 → 기술/운영 → 로드맵
        
-  **핵심 메시지 정리**
    
    - “왜 이 MVP가 사용자에게 가치가 있는가?” 중심으로 스토리라인 설계
        
-  **PPT 비주얼 구성**
    
    - 90~120초 데모 포함, 7~10페이지 내외 구성
        
    - 그래프/데이터/UX 흐름 시각화
        

---

### 4️⃣ 기술/운영 문서 정리

-  **운영 문서 정리**
    
    - README(설명/빌드법), .env.example, 시드 스크립트, 배포 태그 등 정리
        
-  **운영/보안 고려사항**
    
    - 샘플/가명 데이터 사용
        
    - 민감정보 노출 방지 검증
        
-  **기술 로그 백업**
    
    - API 호출 기록, 주요 오류 로그, 트래픽 대응 계획
        

---

### 5️⃣ 영상 및 시연 품질 향상

-  **데모 화면 구성**
    
    - 오류 메시지, 대시보드, 로그인 인터랙션 등 핵심 UI 강조
        
-  **프레젠테이션 리허설**
    
    - 시간 측정 (7–10분 발표 목표)
        
    - 멘트, 화면 전환, 핵심 포인트 타이밍 점검
        
-  **스크립트와 일치 검증**
    
    - “시연 순서”와 “발표 대사”의 불일치 없는지 확인
        

---

### 6️⃣ 제출용 산출물 정리

-  **MVP 실행본 / 배포 링크**
    
    - 실제 동작 가능한 버전(또는 로컬 실행 패키지)
        
-  **발표 자료 (PPT)**
    
    - 시연 영상 / Fallback 영상 링크 포함
        
-  **테스트 리포트**
    
    - 수행 테스트 목록, 성공 여부, 발견된 이슈 요약
        
-  **운영 문서 패키지**
    
    - README + 환경설정 파일 + 시드 스크립트
