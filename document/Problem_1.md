## 📋 MVP 정의 및 성공기준 수치화 To-Do List

### 1️⃣ 사용자 핵심 Pain Point 정의

-  **사용자 스토리 작성**
    
    - 형식: _As a [유형], I want [행동] so that [가치]_
        
    - 2~3개의 주요 시나리오 정리
        
-  **문제 정의서 작성**
    
    - 현재 상황 / 해결해야 할 문제 / 성공 시 효과를 구체적으로 기술
        
    - 관련 데이터나 인터뷰 근거 포함
        

---

### 2️⃣ 우선순위 설정

-  **우선순위 기법 적용**
    
    - `MoSCoW`, `RICE` 중 한 가지 선택
        
    - 각 요구사항의 Must / Should / Could / Won’t 정리
        
-  **기능 vs 비기능 요구사항 분리**
    
    - 기능 요구사항 (Feature)
        
    - 비기능 요구사항 (성능, 보안, 안정성 등)
        
-  **우선순위 표 작성**
    
    - IN(이번에 포함할 것) / OUT(이번엔 제외할 것) 명확히 구분
        

---

### 3️⃣ 시장 및 맥락 분석

-  **JTBD 분석**
    
    - 사용자의 “해야 하는 일(Job)” 중심으로 Pain & Gain 도출
        
-  **SWOT 분석**
    
    - 내부 강점/약점, 외부 기회/위협 정리
        
-  **AARRR 퍼널 정의**
    
    - Acquisition → Activation → Retention → Revenue → Referral 단계별 지표 설정
        

---

### 4️⃣ MVP 핵심 시나리오 설계

-  **핵심 사용자 시나리오 작성**
    
    - 시작 조건 → 행동 → 완료 조건
        
    - 예상 사용자 행동 플로우 다이어그램(Optional)
        
-  **페르소나 정의**
    
    - 목표, 맥락, 주요 과업, 제약 사항 포함
        

---

### 5️⃣ KPI 및 성공지표 수치화

-  **KPI 설정**
    
    - 수치형 목표 (예: 가입 전환율 20%, D7 유지율 15%)
        
-  **성과 측정 기준 문서화**
    
    - 측정 방법, 데이터 출처, 검증 주기 명시
        

---

### 6️⃣ PRD 초안 작성 (v1)

-  **PRD 목차 구성**
    
    - 문제정의 → 페르소나 → 시나리오/저니 → 요구사항 목록 → 비기능요구 → KPI/성공기준
        
-  **RICE/MoSCoW 결과 반영**
    
-  **JTBD/SWOT/AARRR 요약 삽입**
    
-  **릴리즈 스코프 명세**
    
    - IN / OUT 항목을 표 형식으로 정리
        

---

### 7️⃣ 일정 기반 MVP 스코프표 & 위험요소 관리

-  **MVP 스코프 표 작성**
    
    - 필수 기능 / 보완 기능 / 데이터 의존성 명시
        
-  **위험요소(리스크) 정의**
    
    - 일정, 기술, 보안, 데이터 위험별로 구분
        
-  **리스크 완화 계획 작성**
    
    - 각 리스크의 대응 전략 기록
